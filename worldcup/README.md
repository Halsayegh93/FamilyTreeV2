# 🏆 توقعات كأس العالم 2026 — World Cup Prediction Site

موقع مستقل (ما له علاقة بتطبيق العائلة) لتوقّع مباريات كأس العالم من **دور الـ32 حتى النهائي**.
كل واحد يكتب اسمه ويتوقّع النتائج، والنقاط تنحسب تلقائياً ويطلع ترتيب عام مشترك بين الجميع.

## كيف يشتغل؟

- صفحة المباريات: كل واحد يكتب اسمه، يحط نتيجة كل مباراة، ويضغط حفظ.
- التوقعات تنحفظ في Supabase ويشوفها الجميع في **الترتيب**.
- المباراة تُقفل تلقائياً عند بدايتها (لو حطيت لها موعد) أو لما المشرف يعتمد نتيجتها.
- المشرف يدخل `admin.html` بكلمة سر، يحدّث أسماء الفرق والنتائج الرسمية.

## نظام النقاط

| الحالة | النقاط |
|---|:---:|
| النتيجة صحيحة للفريقين (مثال 2-1 = 2-1) | **5** |
| الفائز صحيح + نتيجة الفريق الفائز صحيحة (والخسران غلط) | **3** |
| عرفت الفائز فقط بدون النتيجة | **1** |
| الفائز غلط | **0** |

> أعلى مستوى ينطبق هو اللي يُحتسب (مو تجميع). تقدر تغيّر القيم من `config.js`.

## الفرق والجدول

- **دور الـ32 معبّى تلقائياً** بالفرق وأعلامها ومواعيد وملاعب المباريات.
- **الأدوار اللي بعده تتعبّى لحالها**: أول ما المشرف يعتمد نتيجة، الفائز ينتقل
  تلقائياً للمباراة اللي بعدها (والخاسر في نصف النهائي يروح لمباراة المركز الثالث).
- يعني المشرف وظيفته الوحيدة: **يحط النتيجة**. وفي حالة التعادل بالأدوار الإقصائية
  يختار الفائز بالركلات.
- مطابقات دور الـ32 مبدئية وتنحدّث من لوحة التحكم لو احتجت بعد قرعة الأدوار النهائية.

## الإعداد (مرة وحدة)

1. **سوِّ مشروع Supabase مجاني** على https://supabase.com
2. افتح **SQL Editor** ← New query، الصق محتوى `db.sql` كامل واضغط **Run**.
3. **غيّر كلمة سر المشرف**: في `db.sql` دوّر على `CHANGE_ME` وبدّل `'1234'` بالرقم اللي تبيه (موجود في مكانين — خليهم نفس الرقم).
4. من **Project Settings → API** خذ:
   - `Project URL`
   - `anon` / `publishable` key
   وحطهم في `config.js`.
5. افتح `index.html` في المتصفح — أو ارفع المجلد على استضافة ثابتة.

### النشر (اختياري)

أي استضافة مواقع ثابتة تشتغل (الملفات HTML/CSS/JS فقط):
- **GitHub Pages**: فعّلها على المجلد، الرابط يصير `…/worldcup/`
- **Netlify / Vercel / Cloudflare Pages**: اسحب المجلد فقط

## الملفات

| الملف | الوظيفة |
|---|---|
| `index.html` | صفحة التوقعات + الترتيب |
| `admin.html` | لوحة المشرف (نتائج + أسماء الفرق) |
| `app.js` | منطق مشترك: Supabase + احتساب النقاط |
| `config.js` | بيانات Supabase + نظام النقاط |
| `styles.css` | التصميم (عربي RTL، وضع داكن) |
| `db.sql` | إنشاء الجداول والدوال (شغّله مرة في Supabase) |

## الأمان

- مفتاح Supabase العلني (anon) آمن للنشر — الحماية عبر **Row Level Security**:
  الجميع يقرأ فقط، وكل الكتابة تمر عبر دوال `SECURITY DEFINER` تتحقق من القواعد.
- كلمة سر المشرف تُفحص **داخل قاعدة البيانات** (في `db.sql`)، الموقع ما يثق بأي كلمة سر من المتصفح.
- التوقعات تُقفل من السيرفر نفسه بعد بداية المباراة، فما أحد يقدر يغيّر توقعه بعد ما تبدأ.
